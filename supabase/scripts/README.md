# Supabase quick-apply scripts

## `all-migrations.sql`

A single concatenation of every migration in `supabase/migrations/`. Use this
when you don't want to install the Supabase CLI:

1. Open https://supabase.com/dashboard → your project → SQL Editor → New query.
2. Paste the entire contents of `all-migrations.sql`.
3. Click **Run**. ~2 seconds. Schema, RLS, triggers, and RPC are now live.

> Note: this file is regenerated by hand whenever migrations change. For real
> deploys prefer `supabase db push`, which respects migration order and
> records `supabase_migrations.schema_migrations`.

## After applying

In the Supabase dashboard:

1. **Authentication → Providers** → enable **Phone (SMS)**, point at your
   Twilio / MessageBird credentials. Set the message template to:
   `הקוד שלך ל-שכנה הוא: {{ .Code }}`
2. **Authentication → Providers** → enable **Google** with your OAuth client.
3. **Project Settings → API** → copy the URL + anon + service-role keys into
   your `.env.deploy`.
4. **Project Settings → Functions** → set secrets for the edge functions:
   `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_CONNECT_ACCOUNT_ID`,
   `SUPABASE_SERVICE_ROLE_KEY`.
5. Run `supabase functions deploy <fn>` for each function in
   `supabase/functions/`.
